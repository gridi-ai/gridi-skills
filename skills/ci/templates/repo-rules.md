## Git Configuration

### Branch Strategy
- **Main Branch**: `main`
- **Feature Branch**: `feature/{backlog-keyword}`
- **Bugfix Branch**: `bugfix/{issue-id}`
- **Hotfix Branch**: `hotfix/{description}`

### Protected Branches
- `main`: PR required, CI checks required, squash merge only

## Commit Convention

### Format
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Types
| Type | Description |
|------|-------------|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation changes |
| `style` | Code style changes (no functional changes) |
| `refactor` | Refactoring |
| `test` | Add/modify tests |
| `chore` | Build, configuration file changes |
| `perf` | Performance improvement |
| `ci` | CI configuration changes |

### Examples
```
feat(auth): add JWT authentication
fix(api): handle null response from external service
docs(readme): update installation instructions
refactor(user): extract validation logic to separate module
```

## Lint Rules

### JavaScript/TypeScript
```bash
# Lint check
npm run lint

# Auto-fix
npm run lint:fix

# Prettier formatting
npm run format
```

### Python
```bash
# Ruff lint
ruff check .

# Ruff auto-fix
ruff check --fix .

# Black formatting
black .
```

### Pre-commit Hooks
```bash
# Install pre-commit
pre-commit install

# Manual execution
pre-commit run --all-files
```

## Test Commands
