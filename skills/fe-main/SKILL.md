# Frontend Main Developer

Develops the frontend based on design specs (design-spec.md), Figma designs (if available), and OpenAPI specs.

## ⛔ Generated API Client Must Be Used (GRIDI Required Pattern)

> **⚠️ Required**: All API calls must exclusively use the generated API client.

### GRIDI API Code Generation Structure

```
docs/openapi.yaml                          # OpenAPI spec (single source of truth)
    ↓ npm run api:generate:fe
frontend/src/api/generated/                 # Auto-generated by openapi-generator-cli (do not modify!)
    ├── api.ts                              # {Module}Api classes
    ├── models/                             # DTO types
    └── configuration.ts                    # API configuration
    ↓
frontend/src/api/client.ts                  # Shared axios instance + per-module client exports
    ├── authApiClient                       # AuthApi
    ├── usersApiClient                      # UsersApi
    ├── explorationProjectsApiClient        # ExplorationProjectsApi
    ├── explorationApiClient                # ExplorationApi
    ├── paymentApiClient                    # PaymentApi
    ├── adminApiClient                      # AdminApi
    └── ... (all modules)
```

### Core Rules

1. **⛔ Direct axios/fetch calls prohibited**: Direct calls like `axios.get()`, `fetch()`, `apiAxios.get()` are prohibited
2. **⛔ Use only generated clients**: Call in the form `{module}ApiClient.{method}()`
3. **⛔ Do not modify generated code**: Do not directly edit files under `frontend/src/api/generated/`
4. **⛔ Use generated types too**: `import type { ... } from '@/api/client'` or `'@/api/generated/models'`

### Usage Example

```typescript
// ✅ GOOD: Using generated API client
import { explorationProjectsApiClient } from '@/api/client';
import type { ExplorationProject } from '@/api/client';

const { data } = await explorationProjectsApiClient.getExplorationProject({ id });

// ✅ GOOD: Using generated types
import type { CreateExplorationProjectRequestDto } from '@/api/client';

// ❌ BAD: Direct axios call
import { apiAxios } from '@/api/client';
const { data } = await apiAxios.get(`/exploration-projects/${id}`);

// ❌ BAD: Direct fetch call
const res = await fetch(`/api/exploration-projects/${id}`);

// ❌ BAD: Local interface definition (when generated type exists)
interface ExplorationProject { id: string; name: string; ... }
```

### When the OpenAPI Spec Changes

```bash
# 1. Regenerate code after spec modification (BE + FE simultaneously)
npm run api:generate

# 2. Regenerate frontend only
npm run api:generate:fe

# 3. Check for broken imports with type check
cd frontend && npx tsc --noEmit
```

### When Adding a New API Module

1. Add new tag + endpoints to `docs/openapi.yaml`
2. Run `npm run api:generate:fe`
3. Add new client instance to `frontend/src/api/client.ts`:
```typescript
import { NewModuleApi } from '@/api/generated/api';
export const newModuleApiClient = new NewModuleApi(configuration, API_BASE_URL, apiAxios);
```

### Internationalization (i18n) Required

- All UI text must use the `useTranslations()` hook
- Translation files: `frontend/messages/{ko,en}.json`
- Hardcoded text is prohibited

// components/forms/login-form.tsx
'use client';

import { useState } from 'react';
import { useRouter } from 'next/navigation';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';
import { Button } from '@/components/ui/button';
import { Input } from '@/components/ui/input';
import { useAuth } from '@/hooks/use-auth';
import { loginSchema } from '@/lib/schemas/auth';

type LoginFormData = z.infer<typeof loginSchema>;

export function LoginForm() {
  const router = useRouter();
  const { login, isLoading } = useAuth();
  const [error, setError] = useState<string | null>(null);

  const {
    register,
    handleSubmit,
    formState: { errors },
  } = useForm<LoginFormData>({
    resolver: zodResolver(loginSchema),
  });

  const onSubmit = async (data: LoginFormData) => {
    try {
      setError(null);
      await login(data);
      router.push('/dashboard');
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Login failed.');
    }
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)} className="space-y-4">
      {error && (
        <div className="p-3 text-sm text-red-500 bg-red-50 rounded-md">
          {error}
        </div>
      )}

      <div className="space-y-2">
        <label htmlFor="email" className="text-sm font-medium">
          Email
        </label>
        <Input
          id="email"
          type="email"
          placeholder="email@example.com"
          {...register('email')}
          error={errors.email?.message}
        />
      </div>

      <div className="space-y-2">
        <label htmlFor="password" className="text-sm font-medium">
          Password
        </label>
        <Input
          id="password"
          type="password"
          placeholder="••••••••"
          {...register('password')}
          error={errors.password?.message}
        />
      </div>

      <Button type="submit" className="w-full" disabled={isLoading}>

          Sign Up
        </a>
      </div>

#### 4.2 Custom Axios Instance

```typescript
// src/api/custom-instance.ts
import Axios, { AxiosRequestConfig, AxiosError } from 'axios';
import { useAuthStore } from '@/stores/auth-store';

export const AXIOS_INSTANCE = Axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000/api/v1',
});

// Request interceptor: auto-attach token
AXIOS_INSTANCE.interceptors.request.use((config) => {
  const token = useAuthStore.getState().token;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor: handle token expiration
AXIOS_INSTANCE.interceptors.response.use(
  (response) => response,
  async (error: AxiosError) => {
    if (error.response?.status === 401) {
      useAuthStore.getState().clearToken();
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

// Custom instance function for orval
export const customInstance = <T>(config: AxiosRequestConfig): Promise<T> => {
  const source = Axios.CancelToken.source();
  const promise = AXIOS_INSTANCE({
    ...config,
    cancelToken: source.token,
  }).then(({ data }) => data);

  // @ts-expect-error cancel property for react-query
  promise.cancel = () => {
    source.cancel('Query was cancelled');
  };

  return promise;
};

export default customInstance;
```

#### 4.5 Using Generated Hooks in Components

```typescript
// hooks/use-auth.ts - Wrapping generated hooks to add business logic
import { useQueryClient } from '@tanstack/react-query';
import { useLogin, useLogout, useGetMe } from '@/api/generated/auth/auth';
import { useAuthStore } from '@/stores/auth-store';

export function useAuth() {
  const queryClient = useQueryClient();
  const { setToken, clearToken, token } = useAuthStore();

  // Using generated hooks
  const loginMutation = useLogin();
  const logoutMutation = useLogout();
  const { data: user, isLoading: isLoadingUser } = useGetMe({
    enabled: !!token,
  });

  const login = async (data: Parameters<typeof loginMutation.mutateAsync>[0]) => {
    const result = await loginMutation.mutateAsync(data);
    setToken(result.accessToken);
    queryClient.invalidateQueries({ queryKey: ['auth', 'me'] });
    return result;
  };

  const logout = async () => {
    await logoutMutation.mutateAsync();
    clearToken();
    queryClient.clear();
  };

  return {
    user: user?.data,
    isLoading: loginMutation.isPending,
    isLoadingUser,
    isAuthenticated: !!token,
    login,
    logout,
    error: loginMutation.error,
  };
}
```

  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {

  ({ className, variant, size, isLoading, children, ...props }, ref) => {
    return (

### 6. State Management

// stores/auth-store.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface AuthState {
  token: string | null;
  setToken: (token: string) => void;
  clearToken: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      token: null,
      setToken: (token) => set({ token }),
      clearToken: () => set({ token: null }),

## ⛔ Lint Check Required Principle

> **Warning**: After writing/modifying code, you must run a lint check and fix all errors before finishing.

### Required Execution Order

```
1. Code writing/modification complete ──────────────── ✅
           │
           ▼
2. Run lint auto-fix (--fix) ───────────────────────── ✅ Required
           │
           ├── All errors fixed ──▶ Proceed to step 3
           │
           └── Unfixed errors remain ──▶ Fix manually, then re-run step 2
           │
           ▼
3. Lint check (confirm 0 errors) ──────────────────── ✅ Required
           │
           ▼
4. TypeScript type check ──────────────────────────── ✅ Required
           │
           ▼
5. Build verification (optional) ──────────────────── Recommended
           │
           ▼
6. Task complete
```

### Lint Commands

```bash
# Step 1: Attempt auto-fix (always run first!)
npm run lint -- --fix
# Or for Next.js
npx next lint --fix

# Step 2: Check remaining errors
npm run lint

# Step 3: TypeScript type check
npx tsc --noEmit

# Step 4: Build verification (recommended)
npm run build
```

### Lint Commands by Framework

| Framework | Lint Auto-Fix | Lint Check | Type Check |
|-----------|--------------|-----------|------------|
| Next.js | `npx next lint --fix` | `npx next lint` | `npx tsc --noEmit` |
| React (Vite) | `npm run lint -- --fix` | `npm run lint` | `npx tsc --noEmit` |
| Vue | `npm run lint -- --fix` | `npm run lint` | `npx vue-tsc --noEmit` |
| Svelte | `npm run lint -- --fix` | `npm run lint` | `npx svelte-check` |

### Prettier Integration (If Available)

```bash
# Prettier auto-fix
npx prettier --write .

# Prettier check
npx prettier --check .

# Run ESLint + Prettier together
npm run lint -- --fix && npx prettier --write .
```

### Handling Errors That Cannot Be Auto-Fixed

Errors not fixed by `--fix` must be **fixed manually**:

```typescript
// ❌ Cannot be auto-fixed: using any type
const data: any = fetchData();  // ESLint: Unexpected any

// ✅ Manual fix: specify correct type
interface UserData { id: string; name: string; }
const data: UserData = fetchData();
```

```typescript
// ❌ Cannot be auto-fixed: missing React Hook dependency
useEffect(() => {
  fetchUser(userId);
}, []);  // ESLint: React Hook useEffect has missing dependency: 'userId'

// ✅ Manual fix: add dependency
useEffect(() => {
  fetchUser(userId);
}, [userId]);
```

### Completion Criteria

- ⛔ **Cannot complete task if lint errors remain**
- ⛔ **Cannot complete task if TypeScript errors remain**
- ✅ Task is complete only when `npm run lint` and `npx tsc --noEmit` return 0 errors

## Development Principles

### 1. Lint Check Required
- Always run `npm run lint -- --fix` after writing code
- Manually fix errors that cannot be auto-fixed
- Also verify type errors with `npx tsc --noEmit`
- Task can only be completed with 0 lint/type errors

### 2. Generated API Client Must Be Used
- **⛔ Direct axios/fetch calls prohibited**: Must use `{module}ApiClient.{method}()` format
- **Do not manually define types**: Use only auto-generated types from the OpenAPI spec (`@/api/client` or `@/api/generated/models`)
- **Do not modify generated code**: Never edit files in the `api/generated/` folder
- **Adding clients**: Add new module instances to `frontend/src/api/client.ts`

```bash
# Must run when the OpenAPI spec changes
npm run api:generate
```

### 3. Component Separation
- Separate components by concern
- Reusable UI components

### 4. Absolute Paths Required
- **Relative paths prohibited**: Imports in `../../../` format are prohibited
- **Absolute path root**: Set `@` to the service root (e.g., `frontend/src`)
- All imports must use the `@/` prefix

```typescript
// ❌ Prohibited: relative paths
import { Button } from '../../../components/ui/button';
import { useAuth } from '../../hooks/use-auth';

// ✅ Required: absolute paths
import { Button } from '@/components/ui/button';
import { useAuth } from '@/hooks/use-auth';
```

#### tsconfig.json Configuration (Next.js/React)

```json
// frontend/tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

> **Note**: Next.js supports `@/*` paths by default. It is automatically configured when installing with `create-next-app`.

#### Additional Vite Project Configuration

```typescript
// frontend/vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
});
```

### 5. TypeScript Enum Prohibited - Use as const
- Use `as const` objects instead of `enum` in the frontend as well
- API-generated types use OpenAPI enums and should be used as-is
- Apply the `as const` pattern when defining frontend-specific enumerations

### 6. Type Safety
- Use auto-generated types from the OpenAPI spec (`@/api/model`)
- Use Zod only for form input validation (separate from API types)
- Keep generated types and Zod schemas in sync

### 7. React Query Usage Patterns
- Manage server state with React Query
- Manage client state with Zustand
- Use `queryClient.invalidateQueries()` for cache invalidation

### 8. Accessibility
- Semantic HTML
- ARIA labels
- Keyboard navigation

### 9. Responsive Design
- Mobile-first approach
- Leverage Tailwind breakpoints
